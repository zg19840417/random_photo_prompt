## Project

`random_photo_prompt` is a ComfyUI custom node that generates randomized portrait prompt text and writes that text into connected CLIP text encode nodes before queue execution.

## Active Node Behavior

The node exposes:

- `scale`: user-facing options 一档, 二档, 三档, 四档, mapped internally to normal, bold, bold_no_outfit, nsfw.
- `shot`: user-facing options 头部, 上半身, 半身, 大半身, 全身, mapped internally to shoulder-and-above head shot, chest-and-above upper body, waist-and-above half body, calf-and-above large half body, and full body.
- `use_pregenerated_prompt`: fixed-prompt cache flag. When enabled and `cached_prompt` exists, queue execution reuses the cached prompt and does not regenerate. When disabled, queue execution generates a fresh prompt through the selected `prompt_rule` before serializing the workflow.
- `auto_resolution`: when enabled, desktop queue preparation applies the inferred width and height to current graph image-size or empty-latent widgets; when disabled, desktop generation keeps the user's manually configured graph resolution. Mobile generation still patches its API workflow from inferred resolution.
- `prompt_rule`: desktop prompt-generation rule selector. `规则1` is the original random photo logic; `规则2` is the assistant-style offline keyword-package expansion logic. Both rules must respect the selected `scale` and `shot`; rule2 changes prompt-writing style, not the meaning of the UI controls. The node keeps a single `预生成提示词` button. Manual pregeneration always refreshes through the selected rule; queue-time generation uses the selected rule only when fixed prompt is disabled or no cached prompt exists.
- Hidden cache widgets: `cached_prompt`, `cached_signature`, `cached_aspect`.

## Screenshot-Driven Prompt QA

When a user provides a screenshot of a generated prompt, treat every issue as a common rule problem:

- Verify the same text surface the user sees. Desktop node previews and mobile/web pages may display `display_prompt`, while generation may use `positive_prompt` or `compact_prompt`; all surfaces must share final cleanup.
- Do not rely on pool-only inspection. Instantiate complete prompts, split them into sentences/clauses, and review whether each sentence is concrete, drawable, and natural.
- Trace the source before patching: source pool text, generated data, postprocess order, rebuilt prompt text, display text, and whether the running ComfyUI process has loaded the newest files.
- If local validation is needed, kill the actual process bound to port 8188 and start a fresh local ComfyUI process before API checks. A stale process invalidates any self-check.
- Add screenshot failures to the bad-example audit set when possible. Typical failures include abstract purpose words, explanation words, unclear pronouns such as "观众", repeated gaze/lip clauses, and stitched phrases such as "完整S线被姿态拉开".
- Real display-prompt review must also scan for evaluative intensifiers and doubled words such as "更直接", "更强", "更明显", "眼神眼神", "嘴角嘴角", or "身体身体"; these usually indicate generated text is judging the image instead of describing drawable visual facts.
- Pose sentences longer than 40 Chinese characters must have clear comma-separated action boundaries. If body, hands, gaze, lips, and expression are fused into one unpunctuated chain, the prompt fails even if no known bad word appears.
- Passing QA means both offline audit and real local API display-prompt sampling pass. Offline `tools/audit_generated_prompts.py` alone is not enough for screenshot-facing fixes.
